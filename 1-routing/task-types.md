# Task Types Routing

## Quick answer
Read: L0 -> L1 -> relevant L2

## Artefact ingestion / source doc synthesis
Read: L0 -> L1 -> relevant L2
Write: L5 source note -> optional L4 context module -> refresh L3/L2 as needed

## PRD / spec
Read: L0 -> L1 -> relevant L2 -> L3 -> L4

## Prioritisation / roadmap
Read: L0 -> L1 -> L2 (strategy + user + product) -> L3 -> L4

## Validation challenge
Read: L0 -> L1 -> L2 -> L3 -> L4 -> L5

## Deep audit / dispute
Read: L0 -> L1 -> L2 -> L3 -> L4 -> L5 (+L6 if required)
