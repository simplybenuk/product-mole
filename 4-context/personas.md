# User Personas

Use this file to maintain a small set of living user personas that improve over time as Mole inbox material is synthesised.

Personas should be evidence-backed, provisional, and useful for product decisions. Prefer updating an existing persona when new inbox material clarifies the same user type; create a new persona only when the input points to a meaningfully distinct segment, role, motivation, workflow, or constraint.

## Inbox synthesis rules

When synthesising `6-raw/inbox/`:

1. Check whether each substantive user/customer signal is relevant to an existing persona.
2. Update the matching persona when the signal changes or strengthens goals, behaviours, pains, constraints, objections, success criteria, channels, buying influence, or adoption triggers.
3. Create a new persona section when repeated or high-confidence evidence points to a distinct user type that is not represented here.
4. Add source links to the originating inbox item, evidence note, source document, interview, transcript, or signal cluster.
5. Preserve uncertainty. Mark weak or early signals as hypotheses instead of facts.
6. Update `2-summaries/user-summary.md` when persona changes materially affect the overall user understanding.
